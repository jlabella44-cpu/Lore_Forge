## Gotchas noticed along the way

- **`.env` leaks in pytest failures.** When a Settings instance
  ends up in a pytest assertion dump, every secret on
  `backend/app/config.Settings` prints in the repr. `IMAGE_PROVIDER`
  overridden in the local `.env` causes
  `test_settings_router.test_put_providers_updates_subset` to fail
  with an otherwise clean repr. Rotate + use a test-specific .env or
  patch the assertion.
- **Pre-existing Windows failures.** 3 tests in `test_paths.py` +
  `test_db_url.py` assume POSIX absolute paths and fail on Windows.
  They're in the baseline; ignore when running locally.
- **jinja2 + keyring** needed to be `pip install`ed into the
  `backend/.venv` ‚Äî not sure why; requirements.txt pins them but the
  venv was stale.
- **Parallel bash tool runs share cwd**. I parallelised `pytest` +
  `tsc` a few times and one of them ran from the wrong directory.
  Serialise when the commands cd.
- **Rebase-before-push** is required ‚Äî the A7 work was landed by a
  parallel session and my duplicate commit blew up on rebase. Short
  rebase windows + fetch-then-rebase catches this.
